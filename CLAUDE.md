## Project Overview

Cura is an AI-powered robotic feeding assistant for hospital patients, built for HackStorm 2.0 (May 22-24, 2026). It controls a Piper robot arm via CAN bus to deliver food/water to patients, coordinated by a FastAPI server and a T5AI touchscreen board.

## Common Commands

All Python commands use `uv`:

```bash
# Install dependencies
uv sync

# Run the full system (requires waypoints.json and CAN hardware)
uv run python -m cura.main

# Environment overrides
CURA_ARM_SPEED=30 CURA_WAYPOINTS_FILE=waypoints.json uv run python -m cura.main

# Run all tests (no hardware required)
uv run python -m pytest tests/ -v

# Run a specific test file
uv run python -m pytest tests/test_interface.py -v

# Record arm waypoints interactively (requires CAN hardware)
python3 piper_sdk/record_waypoints.py --can_port can0 --output waypoints.json

# Replay recorded waypoints
python3 piper_sdk/replay_waypoints.py --can_port can0 --file waypoints.json --speed 50
```

### CAN setup (run once per boot, before any arm commands)

```bash
sudo ip link set can0 up type can bitrate 1000000
```

### T5AI firmware build and flash

```bash
cd t5ai
cmake -B build -DTUYA_SDK_PATH=$TUYA_SDK_PATH
cmake --build build -j$(nproc)
pip install tuya-iot-tool
tuya-flash --port /dev/ttyUSB0 --firmware build/cura_t5ai.bin
```

Edit `t5ai/src/cura_config.h` with Wi-Fi credentials and laptop IP before flashing.

## Architecture

The system has four layers that communicate at runtime:

```
Web dashboard (Lovable) ──┐
T5AI board (HTTP poll)   ──┼──► FastAPI server (CuraServer)
Operator keyboard ─────────┘         │
                                      ▼
                              Orchestrator (main.py)
                                      │
                         ┌────────────┼────────────┐
                         ▼            ▼             ▼
                    Arm control    Vision/ML    Config/Settings
                    (piper_sdk)  (YOLO+MP)    (env vars)
```

### Python package: `src/cura/`

The main package is split by member ownership:

- **`src/cura/main.py`** — State machine orchestrator. States: `IDLE → APPROACHING → GRASPING → LIFTING → DELIVERING → DRINKING → RETRACTING`. SPACE triggers pickup; ESC is emergency stop.
- **`src/cura/arm/`** — Piper arm control via `piper_sdk`. `controller.py` wraps `C_PiperInterface_V2`; `trajectories.py` handles teaching and playback; `safety.py` enforces joint limits. Reads waypoints from `waypoints.json`.
- **`src/cura/interface/`** — FastAPI server (`CuraServer`) with `GET /status`, `POST /command`, `GET /ws` (WebSocket), `GET /meal`, `GET /health`. Models (`SystemState`, `SystemStatus`, `PatientCommand`) are in `models.py` — use them as-is.
- **`src/cura/vision/`** — YOLOv8n food/bottle detection (`detector.py`), camera capture (`camera.py`), MediaPipe mouth tracking (`mouth_tracker.py`). Not wired into V1 main loop.
- **`src/cura/config/settings.py`** — All tunable parameters; overridable via env vars (`CURA_ARM_SPEED`, `CURA_WAYPOINTS_FILE`, etc.).

### `piper_sdk/` (vendored SDK + scripts)

Vendored copy of the Piper arm SDK source plus Cura-specific scripts:
- `record_waypoints.py` — Interactive teach-mode recorder. Puts arm in drag-teach mode (`MotionCtrl_1(0,0,1)`), reads joint angles via `GetArmJointMsgs()`, saves to JSON.
- `replay_waypoints.py` — Replays a JSON waypoints file using `JointCtrl()` at ~50 Hz, polling `GetArmStatus().arm_status.motion_status` to detect arrival.
- `*.json` files — Recorded task trajectories (joints in 0.001-degree units, gripper in 0.001-mm units).
- `server.py` — Standalone socket trigger server for quick arm testing (port 9999, "GO" command).

### `t5ai/` (embedded firmware)

TuyaOpen C firmware for the Tuya T5AI board (ARM Cortex-M33, LVGL v9 display):
- Polls `GET /status` every 500 ms, renders `state_label` on screen.
- START button → `POST /command {"action":"start_feeding","source":"t5ai"}`
- EMERGENCY STOP → `POST /command {"action":"emergency_stop","source":"t5ai"}`

### `waypoints.json`

JSON file at repo root containing named arm poses. Structure:
```json
{"name": "task", "created": "...", "waypoints": [{"label": "home", "joints": [j1..j6], "gripper_angle": N, "gripper_effort": N}]}
```
Joint values are in **0.001-degree units** (multiply by `1e-3` to get degrees). Generated by Member B via `record_waypoints.py`.

## Git Workflow

- Branch from `master` as `member-[A-E]/[feature]`
- Never push directly to `master`; open a PR and tag the relevant member
- Ownership boundaries (honour system):
  - Member A: `src/cura/vision/`, `tests/test_vision.py`
  - Member B: `src/cura/arm/`, `tests/test_arm.py`, `waypoints.json`
  - Member C: `src/cura/main.py`, `src/cura/config/settings.py`, `tests/test_integration.py`
  - Member D: `src/cura/interface/`, `t5ai/`
  - Member E: web dashboard (separate repo), `prints/`, pitch deck

## Key Cross-Boundary Contracts

- `CuraServer` API (Member D → Member C): `server.run()`, `server.update_state(state, estop_active=False)`, `server.get_next_command()`, `server.stop()`
- WebSocket message format (Member D → Member E): `{"state", "state_label", "message", "is_moving", "estop_active", "timestamp"}`
- Mouth-tracking coordinates (Member A → Member C, V2): injected via `threading.Event` + queue into orchestrator
